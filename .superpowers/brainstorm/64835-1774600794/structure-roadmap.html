🗓 分阶段实现路线图

1 Phase 1 · 基础重构（无新功能，保持现有能力）

  • 提取 Provider 接口，将 engine.go 中的 OpenAI 调用迁移到 pkg/provider/openai.go
  • 将 ClawEngine 重构为 BaseAgent，实现 Agent 接口
  • 将 CLI readline 逻辑提取为 CLIChannel，实现 Channel 接口
  • 引入 Orchestrator 骨架，串联 CLIChannel → Orchestrator → CoderAgent
  • 全部现有测试通过，用户无感知变化

2 Phase 2 · Memory 层

  • 实现 pkg/memory/store.go，自动建表（IF NOT EXISTS）
  • 实现 Session Memory（72h TTL）+ Long-term Fact Store + FTS5 索引
  • 实现 pkg/memory/prompt.go：记忆 → system prompt 注入
  • 注册 remember / forget / recall 三个工具
  • Orchestrator 在每次会话开始时注入记忆、结束后异步更新 Session Memory

3 Phase 3 · 多 Agent 专化

  • 实现 ReviewerAgent 和 ResearcherAgent（不同 system prompt + 工具子集）
  • 实现 router.go 意图路由（关键词匹配优先，兜底 CoderAgent）
  • 注册 dispatch_agent 工具，支持 CoderAgent 调起子 Agent

4 Phase 4 · Telegram Channel

  • 实现 pkg/channel/telegram/bot.go，Long Polling + 白名单鉴权
  • 实现附件下载（图片/文档 → 本地临时文件）
  • 实现流式刷新（edit_message 每 1.5s）+ 任务完成新消息通知
  • 实现 /reset /memory /status /cancel 命令
  • main.go 支持 --channel telegram 启动参数
改动影响总结：
🟢 完全不动：pkg/tools/（13个工具）、pkg/ui/、现有测试
🔵 小幅扩展：pkg/tools/registry.go（+4行注册）、cmd/agent/main.go（替换为 Channel 启动）
🟡 重构：pkg/agent/engine.go → pkg/agent/base.go（逻辑保留，接口包装）
🟡 新增：pkg/channel/、pkg/orchestrator/、pkg/memory/（~1500行新代码）
