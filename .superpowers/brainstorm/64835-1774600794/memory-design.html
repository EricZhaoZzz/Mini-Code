Memory 层详细设计

SQLite Schema + 三类记忆的读写逻辑

📁 存储位置

// 全局数据库（用户偏好、对话摘要） ~/.mini-code/memory.db // 项目级数据库（每个工作区一个） <workspace>/.mini-code/project.db

🗄️ SQLite Schema

-- 项目记忆表（project.db） CREATE TABLE project_memory ( id INTEGER PRIMARY KEY, key TEXT UNIQUE, -- "tech_stack","code_style","important_paths" value TEXT, updated_at DATETIME ); -- 用户偏好表（memory.db） CREATE TABLE user_preference ( id INTEGER PRIMARY KEY, key TEXT UNIQUE, -- "language","verbosity","code_style" value TEXT, updated_at DATETIME ); -- 对话摘要表（memory.db） CREATE TABLE conversation_summary ( id INTEGER PRIMARY KEY, workspace TEXT, -- 关联到哪个项目 summary TEXT, -- 压缩后的摘要内容 msg_count INTEGER, -- 压缩自多少条消息 created_at DATETIME );

🧠 三类记忆详细说明

① 项目记忆（ProjectMemory）

存什么：技术栈（Go/React）、编码风格（tab缩进/驼峰命名）、关键路径（main入口、配置文件位置）、重要决策（"不要用X库"）
何时读：每次会话开始，注入 system prompt 的"项目背景"部分
何时写：Agent 完成任务后，LLM 判断是否有新的项目信息值得记录（异步调用轻量LLM提取）
READ: 会话开始时 WRITE: 任务完成后异步 LLM 自动提取

② 用户偏好（UserPreference）

存什么：回复语言（中文）、详细程度（简洁/详细）、代码风格偏好、常用操作习惯
何时读：每次会话开始，注入 system prompt 的"用户偏好"部分
何时写：检测到用户明确指令（"以后用中文"、"不要加注释"）时立即写入；也可手动 /remember 命令
READ: 会话开始时 WRITE: 检测到偏好变化时 支持 /remember 命令

③ 对话摘要（ConversationSummary）

存什么：历史对话的压缩摘要（"上周完成了认证模块重构，决定使用JWT"）
何时读：新会话开始时，加载最近 N 条摘要注入上下文
何时写：消息历史超过阈值（默认 40 条）时，自动将最早的 20 条压缩为摘要写入DB，然后从内存中移除
READ: 新会话开始时 WRITE: 消息数超过阈值时 自动压缩，无感知
依赖： 使用 modernc.org/sqlite（纯 Go 实现，无 CGO，跨平台零额外依赖）
并发安全： 每个 DB 连接单独管理，使用 database/sql 连接池，写操作加互斥锁
迁移： 应用启动时自动执行 CREATE TABLE IF NOT EXISTS，无需独立迁移工具
