系统整体架构（分层视图）

Mini-Code → OpenClaw 演进后的完整层次结构

Channel

📡 Channel 层 — 统一输入输出接口

所有外部交互入口。CLI 和 Telegram 实现同一个 Channel 接口，Orchestrator 无需关心来源。

pkg/channel/cli/ pkg/channel/telegram/ pkg/channel/types.go
↓
Orchestrator

🎯 Orchestrator 层 — 会话路由与 Agent 调度

管理跨 Channel 的会话状态，根据任务类型路由到合适的 Agent，协调并发子任务执行。

pkg/orchestrator/router.go pkg/orchestrator/session.go pkg/orchestrator/dispatcher.go
↓
Agent

🤖 Agent 层 — 角色专化执行单元

现有 ClawEngine 重构为 BaseAgent，派生出 CoderAgent（完整工具集）、ReviewerAgent（只读）、ResearcherAgent（搜索+下载）。

pkg/agent/base.go pkg/agent/coder.go pkg/agent/reviewer.go pkg/agent/researcher.go
↓
Memory

🧠 Memory 层 — SQLite 持久化记忆

三类记忆：项目级（技术栈/编码风格）、用户偏好（沟通风格/习惯）、对话摘要（长上下文压缩）。全部存储在 SQLite。

pkg/memory/store.go pkg/memory/project.go pkg/memory/user.go pkg/memory/summary.go
↓
Tool + Provider

🔧 Tool + Provider 层 — 现有基础设施（基本保留）

pkg/tools/ 中的 13 个工具基本不变。pkg/provider/ 补全 Provider 接口抽象，支持未来扩展。

pkg/tools/（13个工具，不变） pkg/provider/interface.go（新增） pkg/ui/（不变）

标注说明： 🟡 NEW = 全新包  |  🔵 EXTEND = 在现有基础上扩展  |  🟢 KEEP = 基本不动
