Memory 层设计 v2（对齐 OpenClaw）

新增 remember / forget 工具 + 3层记忆模型

🏗 三层记忆模型（对齐 OpenClaw）

Layer 1 · Working Memory（当前对话）

消息历史 []ChatMessage，存在 ClawEngine.messages 中，会话结束后清空。现有实现，不变。

Layer 2 · Session Memory（72h 短期）NEW

任务完成后自动提取本次对话要点，写入 SQLite，72小时 TTL 自动过期。下次会话时加载近期 Session 摘要注入 system prompt。对应 OpenClaw 的 session_memory 表。

Layer 3 · Long-term Memory（持久 Fact Store）NEW

Agent 通过 remember() 工具主动写入的事实。按 scope 分三类：project（当前工作区）、user（全局偏好）、global（跨项目通用知识）。永不自动过期，只能 forget() 删除。

🔧 新增工具（注册到 pkg/tools/registry.go）

remember(content, scope, tags?)

content：要记住的事实，自然语言描述
scope："project" | "user" | "global"
tags：可选标签，便于后续检索（如 ["architecture","decision"]）
示例："记住：这个项目使用 pnpm 而不是 npm，因为 monorepo 结构需要 workspace 支持"

forget(memory_id)

memory_id：要删除的记忆 ID（由 remember 返回）
用于纠正错误记忆或删除过时信息

recall(query, scope?)

query：自然语言检索词
scope：可选，限定检索范围（不填则全局搜索）
SQLite FTS5 全文检索，返回相关记忆列表。Agent 在不确定时可主动查询。

🗄 SQLite Schema（最终版）

-- 短期记忆（72h TTL，memory.db） CREATE TABLE session_memory ( id INTEGER PRIMARY KEY, workspace TEXT, content TEXT, created_at DATETIME, expires_at DATETIME -- created_at + 72h ); -- 长期记忆 Fact Store（memory.db） CREATE TABLE long_term_memory ( id INTEGER PRIMARY KEY, scope TEXT, -- "project" | "user" | "global" workspace TEXT, -- scope=project 时关联工作区路径 content TEXT, -- 自然语言事实 tags TEXT, -- JSON 数组 created_at DATETIME ); -- FTS5 全文检索索引 CREATE VIRTUAL TABLE memory_fts USING fts5( content, tags, content=long_term_memory );
与 OpenClaw 的对齐程度：
✅ Working Memory（当前对话上下文）
✅ Session Memory（72h 短期，自动提取）
✅ Long-term Fact Store（LLM 主动 remember/forget/recall）
✅ Scope 分级（project / user / global）
⚡ 简化：OpenClaw 用向量相似度检索，我们用 SQLite FTS5 全文检索（零额外依赖）
