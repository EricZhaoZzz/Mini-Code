② Telegram Bot 集成方式

1

长轮询（Long Polling）：使用 go-telegram-bot-api/telegram-bot-api/v5，无需公网 IP / HTTPS 证书，本机直接运行即可。Bot Token 通过环境变量 TELEGRAM_BOT_TOKEN 配置，白名单用户 ID 通过 TELEGRAM_ALLOWED_USERS 配置（安全隔离）。

2

消息接收：收到文本消息直接转发给 Orchestrator。收到图片/文档附件时，先调用 Telegram API 下载到本地临时目录，再以文件路径传给 Agent（CoderAgent 可 read_file 读取内容）。

3

流式响应：Agent 执行期间，每 1.5 秒 edit_message 刷新同一条消息（避免消息轰炸）。任务完成后发一条新消息（触发手机推送通知）。超过 4096 字符时自动分段发送。

4

并发安全：每个 chat_id 独立 Session，不同用户的任务并发互不干扰。同一 chat_id 串行处理（与 OpenClaw Lane Queue 一致）。

③ Telegram /命令 快捷键

命令功能
/start欢迎消息 + 当前工作区信息
/reset清空当前会话历史（保留长期记忆）
/memory查看当前已记住的项目/用户记忆
/status显示当前正在执行的任务进度
/cancel取消当前正在执行的任务

④ 三类专化 Agent

🔨 CoderAgent — 编写与修改代码

系统 Prompt 强调：代码质量、最小化修改、运行测试验证。触发词：编写/修改/实现/添加/重构/修复 bug。

write_fileread_filelist_files replace_in_filesearch_in_filesrun_shell rename_filedelete_filecopy_file create_directorygit_statusgit_diff git_logdownload_fileremember forgetrecall

👁 ReviewerAgent — 代码审查

系统 Prompt 强调：只读分析、发现问题、给出改进建议（不直接修改代码）。触发词：审查/检查/review/分析代码质量。

read_filelist_filessearch_in_files git_diffgit_loggit_status get_file_inforecall

🔎 ResearcherAgent — 技术调研

系统 Prompt 强调：广泛搜索、整理信息、给出有依据的结论。触发词：调研/了解/搜索/查找/怎么实现X。

read_filelist_filessearch_in_files download_filerun_shellrecall remember
路由策略： Orchestrator 默认使用 CoderAgent。发消息前先用轻量 LLM（或关键词匹配）判断意图，reviewer/researcher 只在明确触发时启用。
子 Agent 调用： CoderAgent 执行复杂任务时可通过 dispatch_agent(role, task) 工具调起 ReviewerAgent 并行验证，结果汇总后返回。
